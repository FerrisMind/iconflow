Ниже — подробный **roadmap + план разработки + Developer Checklist** так, чтобы это можно было прямо отдать команде и начать реализацию.[1][2]

## Roadmap (вехи)
| Фаза | Цель | Результат (DoD-артефакты) | Примерный срок* |
|---|---|---|---|
| 0. Bootstrap | Подготовить репо/CI/политику фич | Каркас crate + CI + правила фич | 1–2 дня |
| 1. Data Contract | Зафиксировать входные данные | JSON Schema + 1–2 pack maps валидируются | 2–4 дня |
| 2. Generator | Генерировать `src/generated/*` | `xtask gen` + детерминированный output | 3–6 дней |
| 3. Core API | Реализовать стабильный API | `fonts()/try_icon()/list()` + ошибки | 3–6 дней |
| 4. Packs | Подключить все иконпаки | `pack-*` модули + feature-гейты | 5–10 дней |
| 5. Examples+Docs | “Сразу понятно как юзать” | README + примеры egui/iced | 2–4 дня |
| 6. Release hardening | Стабилизация и supply-chain | матрица фич в CI + хэши ассетов | 2–5 дней |

\*Оценку лучше делать **story points** через planning poker и не приравнивать points к часам.[3][4][5]
